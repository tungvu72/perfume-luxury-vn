# Workflow

## 1. Diagnose the request type
Choose one:
- planning a calendar
- choosing the next topic
- analyzing a winner
- postmortem on a flop
- scanning competitor patterns

## 2. Start from evidence
Look for:
- current winning topics
- audience fit
- traffic source
- packaging strength
- whether failure came from topic or execution

## 3. Classify the proposed idea
Put it in one lane:
- ranking
- battle
- problem-solving
- opinion/identity

If it fits none, question it before recommending it.

## 4. Score the idea quickly
Check:
- audience fit
- mass clarity
- bottle recognition
- emotional hook
- practical intent
- freshness vs current upload mix

## 5. Recommend with sequencing
Do not suggest isolated ideas only. Also answer:
- what should go next
- what should come after that
- what format should break repetition

## 6. Close the loop
After results come in, log:
- win/flop
- likely cause
- keep / repeat / kill decision
