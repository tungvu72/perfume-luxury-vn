# Skill: Planning & Task Management

## Khi nhận task từ User, luôn viết kế hoạch trước

### Format chuẩn ghi vào TASK_LOG.md:
```
## [YYYY-MM-DD HH:MM] Task: [tên ngắn gọn]
**Yêu cầu:** [copy nguyên văn]
**Phân tích:** [task loại gì, phức tạp không]
**Kế hoạch:**
  [ ] 1. [bước 1 - ai làm - output mong muốn]
  [ ] 2. [bước 2 - ai làm - output mong muốn]
**Deadline:** [ước tính]
**Trạng thái:** Đang thực hiện
```

## Phân loại task nhanh:
| Loại | Dấu hiệu | Giao cho |
|------|---------|---------|
| Kỹ thuật | "sửa", "thêm feature", "fix bug", "code" | Coder |
| Marketing | "viết content", "caption", "SEO", "ads", "phân tích" | Marketer |
| Hỗn hợp | Cần cả code lẫn content | Cả 2 song song |

## Chia task lớn thành task nhỏ:
- Mỗi task nhỏ phải có: AI LÀM + OUTPUT CỤ THỂ + ĐIỀU KIỆN XONG
- Tối đa 5 bước — nếu hơn thì chia thành 2 giai đoạn
- Giao từng bước một, không giao hết cùng lúc

## Update TASK_LOG khi hoàn thành:
```
**Trạng thái:** ✅ Hoàn thành lúc [HH:MM]
**Kết quả:** [tóm tắt ngắn]
```
